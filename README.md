# DormSync 宿舍协同管理平台

基于微信小程序的宿舍协同管理工具，帮助室友轻松管理日常事务。

## 技术栈

- **前端**：uni-app (Vue 3 Composition API) → 微信小程序
- **后端**：Node.js + Express
- **数据库**：MongoDB Atlas (云数据库)
- **认证**：JWT + 微信登录
- **开发工具**：HBuilderX + 微信开发者工具

## 项目结构

```

├── DormSync-server/          # 后端服务

```

## 环境准备（在新电脑上首次配置）

### 1. 安装 Node.js

1. 打开 https://nodejs.org
2. 下载 LTS（长期支持）版本，当前推荐 v18 或 v20
3. 双击安装包，一路点「下一步」即可
4. 安装完成后打开终端（macOS 打开「终端」app，Windows 打开「命令提示符」或「PowerShell」），验证安装：
   ```bash
   node -v    # 应显示 v18.x.x 或 v20.x.x
   npm -v     # 应显示 9.x.x 或 10.x.x
   ```

### 2. 安装 HBuilderX

1. 打开 https://www.dcloud.io/hbuilderx.html
2. 下载对应系统版本（macOS / Windows）
3. macOS：解压后将 HBuilderX 拖入「应用程序」文件夹；Windows：解压到任意目录
4. 打开 HBuilderX，首次启动可能需要安装 uni-app 编译插件，按提示安装即可

### 3. 安装微信开发者工具

1. 打开 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载稳定版（Stable Build），选择对应系统
3. 安装完成后打开，用微信扫码登录
4. 在微信开发者工具中开启「服务端口」：
   - 点击右上角「设置」→「安全设置」→ 打开「服务端口」开关
   - 这一步很重要，HBuilderX 需要通过服务端口与微信开发者工具通信

### 4. 在 HBuilderX 中配置微信开发者工具路径

1. 打开 HBuilderX → 顶部菜单「工具」→「设置」→「运行配置」
2. 找到「微信开发者工具路径」，填写微信开发者工具的安装路径：
   - macOS 默认路径：`/Applications/wechatwebdevtools.app`
   - Windows 默认路径：`C:\Program Files (x86)\Tencent\微信web开发者工具`
3. 点击「确定」保存

## 详细运行步骤

### 第一步：获取项目代码

将整个项目文件夹复制到新电脑上（U盘拷贝、Git clone 等方式均可），确保目录结构如下：

```
你的文件夹/
├── DormSync-server/
├── DormSync-uniapp/
└── README.md
```

### 第二步：安装后端依赖

打开终端，进入后端目录并安装依赖：

```bash
cd 你的文件夹/DormSync-server
npm install
```

等待安装完成，会看到 `added xxx packages` 的提示。这一步会在 `DormSync-server` 下生成 `node_modules` 文件夹。

> 如果 npm install 很慢，可以先切换为国内镜像源：
> ```bash
> npm config set registry https://registry.npmmirror.com
> npm install
> ```

### 第三步：查看本机局域网 IP

```bash
# macOS
ifconfig | grep "inet " | grep -v 127.0.0.1

# Windows
ipconfig
```

找到类似 `192.168.x.x` 或 `172.x.x.x` 的地址，记下来。

> 例如输出 `inet 192.168.1.100`，那你的 IP 就是 `192.168.1.100`

### 第四步：修改前端接口地址

用任意编辑器（HBuilderX、VS Code 等）打开文件：

```
DormSync-uniapp/utils/request.js
```

找到第 7 行左右的 `BASE_URL`，修改为你的 IP：

```javascript
// 修改前
export const BASE_URL = 'http://172.20.42.135:3000'

// 修改后（替换为你自己的 IP）
export const BASE_URL = 'http://192.168.1.100:3000'
```

保存文件。

### 第五步：启动后端服务

在终端中（确保在 `DormSync-server` 目录下）：

```bash
node app.js
```

看到以下输出说明启动成功：
```
✅ MongoDB 连接成功: xxx/dormSync
🚀 DormSync 服务已启动: http://localhost:3000
```

> ⚠️ 如果出现 `MongoDB 连接失败: querySrv ENOTFOUND` 错误：
> 
> 这是 DNS 无法解析 MongoDB Atlas 地址，常见于校园网环境。解决方法：
> 
> **方法一：手机开热点**
> 手机开移动数据热点，电脑连接手机热点后重新运行 `node app.js`
> 
> **方法二：修改系统 DNS**
> - macOS：系统设置 → 网络 → 当前 WiFi → 详细信息 → DNS → 添加 `8.8.8.8` 和 `114.114.114.114`
> - Windows：控制面板 → 网络和共享中心 → 更改适配器设置 → 右键当前网络 → 属性 → IPv4 → 首选 DNS `8.8.8.8`，备用 `114.114.114.114`
> 
> **方法三：使用 VPN**
> 开启 VPN 后重新运行

**注意：后端启动后不要关闭这个终端窗口，保持运行状态。**

### 第六步：启动前端（在微信开发者工具中运行）

1. 打开 HBuilderX
2. 点击菜单「文件」→「打开目录」→ 选择 `DormSync-uniapp` 文件夹
3. 打开后在左侧项目列表中能看到 `DormSync-uniapp`
4. 点击顶部菜单「运行」→「运行到小程序模拟器」→「微信开发者工具」
5. 等待编译完成（底部控制台会显示编译进度）
6. 编译成功后，微信开发者工具会自动打开并显示小程序界面

> 如果微信开发者工具没有自动打开：
> - 检查 HBuilderX 中微信开发者工具路径是否配置正确（见环境准备第4步）
> - 检查微信开发者工具的「服务端口」是否已开启（见环境准备第3步）
> - 尝试先手动打开微信开发者工具，再从 HBuilderX 运行

### 第七步：在微信开发者工具中配置 AppID

1. 微信开发者工具打开后，如果提示输入 AppID：
   - 填入：`wxd49e11e8ebdd3fa2`
   - 或者选择「测试号」
2. 如果已经打开了项目，点击右上角「详情」→「基本信息」→ 确认 AppID 正确

### 第八步：开始使用

小程序加载完成后，你会看到登录页面：

1. 点击「微信一键登录」
2. 在开发者工具中会模拟微信登录（自动完成）
3. 首次登录会跳转到「个人信息编辑」页面 → 填写姓名 → 点击保存
4. 保存后跳转到「宿舍设置」页面 → 选择「创建宿舍」或「加入宿舍」
5. 完成后进入首页，开始使用所有功能

## 真机调试步骤（用手机微信运行小程序）

真机调试可以体验最真实的效果，也是多用户测试的必要方式。

### 前提条件
- 手机和电脑连接同一个 WiFi 网络
- 后端服务已启动（终端中 `node app.js` 正在运行）
- `request.js` 中的 `BASE_URL` 已改为电脑的局域网 IP

### 操作步骤
1. 在微信开发者工具中，点击顶部工具栏的「预览」按钮（不是「真机调试」）
2. 等待编译，完成后会显示一个二维码
3. 打开手机微信，扫描这个二维码
4. 手机上会打开小程序，可以正常使用所有功能

> 预览二维码有效期约 25 分钟，过期后需要重新点击「预览」生成新二维码。

## 多用户测试流程（答辩演示）

多用户协作是本项目的核心功能，以下是完整的多人测试流程。

### 准备工作
- 至少需要 2 部手机（2 个不同的微信号）
- 所有手机和电脑连接同一个 WiFi
- 后端服务已启动

### 用户A（宿舍长）操作流程：

1. **登录**：手机微信扫码进入小程序 → 点击「微信一键登录」
2. **完善信息**：首次登录跳转到个人信息页 → 输入姓名（如"张三"）→ 点击保存
3. **创建宿舍**：跳转到宿舍设置页 → 选择「创建宿舍」→ 输入宿舍号（如"5号楼306"）→ 点击创建
4. **进入首页**：创建成功后自动进入首页，可以看到宿舍号、公告区域、待办列表
5. **邀请室友**：点击底部「更多」tab → 点击「邀请新成员」→ 点击「分享给微信好友」→ 选择室友或宿舍群发送

### 用户B（室友）操作流程：

**方式一：通过分享卡片加入（推荐演示）**
1. 在微信聊天中收到用户A发送的小程序卡片
2. 点击卡片 → 自动打开小程序
3. 点击「微信一键登录」→ 填写个人信息 → 保存
4. 自动跳转到宿舍设置页，宿舍号已预填好 → 点击「加入宿舍」
5. 加入成功，进入首页

**方式二：手动输入宿舍号加入**
1. 手机微信扫码进入小程序 → 登录 → 填写信息
2. 在宿舍设置页选择「加入宿舍」→ 手动输入宿舍号"5号楼306" → 点击加入

### 多人协作功能演示建议：

加入同一宿舍后，可以演示以下协作场景：

1. **费用管理**：用户A 发起一笔费用（如"电费 100元"）→ 用户B 在消息中心收到通知
2. **投票决策**：用户A 发起投票（如"周末聚餐地点"）→ 用户B 参与投票 → 实时看到投票结果
3. **排班管理**：用户A 生成值日排班表 → 所有成员都能看到自己的值日安排
4. **宿舍公告**：用户A 在「更多」→「编辑公告」发布公告 → 用户B 在首页看到公告更新
5. **消息通知**：以上所有操作都会在对方的消息中心产生通知，tabBar 显示未读红点
6. **宿舍长选举**：用户A 在「更多」→「宿舍长选举」将宿舍长转让给用户B

> 注意：在微信开发者工具（模拟器）中无法测试多用户，因为每次登录的 openid 相同。
> 多用户测试必须使用不同微信号在真机上操作。

## 常见问题

**Q: MongoDB 连接失败 `querySrv ENOTFOUND`**
A: DNS 无法解析 MongoDB Atlas 地址。尝试切换网络（手机热点）或修改系统 DNS 为 `8.8.8.8`。

**Q: 小程序请求接口报错 `网络异常`**
A: 检查 `request.js` 中的 `BASE_URL` 是否为当前电脑的局域网 IP，且手机和电脑在同一 WiFi。

**Q: 微信登录失败**
A: 确认 `.env` 中的 `WX_APPID` 和 `WX_SECRET` 正确。在微信开发者工具中需要在项目设置里填写正确的 AppID。

**Q: 头像不显示**
A: 微信小程序真机不支持 HTTP 图片加载。头像通过下载到本地临时文件的方式显示，确保后端 `/uploads` 目录可访问。

**Q: 如何让其他人使用小程序？**
A: 如果使用测试号，需要在测试号管理页面添加体验成员的微信号。正式小程序则可以直接扫码使用。
